# Multimodal SLAM Intelligent Patrol Robot Based on RDK X5

## 1. Project Overview

This project implements an indoor intelligent patrol robot based on the **RDK X5** robotics development platform. It integrates **LiDAR perception, visual sensing, SLAM mapping, Nav2 autonomous navigation, YOLO-based human detection, emergency stop logic, and Web monitoring** into a complete mobile robot system.

The robot uses a **YDLIDAR X3 LiDAR** for 2D environment perception and SLAM mapping, and uses **AMCL localization and the Nav2 navigation framework** for goal navigation and multi-point patrol. An **IMX219-160IR camera** is used to capture patrol images. YOLO detection results are used to identify the `person` class. When a person is detected during patrol, the system publishes an abnormal patrol status, cancels the current Nav2 goal, actively stops the robot, and resumes patrol after the status returns to normal.

The project forms a complete closed-loop workflow:

```text
Environment perception
→ SLAM mapping
→ Localization and path planning
→ Motion execution
→ Human detection
→ Abnormal stop
→ Web monitoring
```

---

## 2. Features

- **SLAM Mapping**: Builds a 2D occupancy grid map using YDLIDAR X3 and slam-toolbox.
- **Autonomous Navigation**: Uses Nav2 for map loading, AMCL localization, path planning, local obstacle avoidance, and motion control.
- **Multi-point Patrol**: Sends predefined patrol goals and navigates through A, B, C, D, and E points in sequence.
- **YOLO Human Detection**: Subscribes to `/hobot_dnn_detection` and detects the `person` class.
- **Abnormal Stop Logic**: Cancels the current Nav2 goal and publishes zero velocity when a person is detected.
- **Patrol Resume**: Continues the unfinished patrol route after `/patrol/status` returns to `Normal`.
- **Web Monitoring**: Displays live image stream, patrol status, alarm information, and basic control buttons.
- **Script-based Demo Workflow**: Provides startup scripts for reproducible demonstration and testing.

## 6. Directory Description

### 6.1 `keycode/`

This directory stores keyboard control, robot debugging, or auxiliary control scripts used during development.

### 6.2 `lidar/`

This directory stores LiDAR and robot model configuration files.

- `X3.yaml`: Parameter file for the YDLIDAR X3 driver.
- `robot_model.yaml`: Robot model and sensor TF configuration file.

During development, the LiDAR direction was calibrated to ensure that `/scan` data matches the real front/back and left/right directions of the robot.

### 6.3 `map/`

This directory stores the final navigation map.

- `patrol_map_v5.pgm`: Occupancy grid map generated by SLAM.
- `patrol_map_v5.yaml`: Map metadata file, including resolution, origin, and thresholds.

`patrol_map_v5` was rebuilt after LiDAR direction calibration and is used for Nav2 localization and patrol navigation.

### 6.4 `nav/`

This directory stores multi-point patrol navigation scripts.

- `patrol_nav_node.py`: Basic multi-point patrol script.
- `patrol_nav_node_safe.py`: Safe patrol version that subscribes to `/patrol/status` and stops on abnormal detection.
- `patrol_nav_node_final.py`: Final demonstration version used for the complete patrol workflow.

The final patrol logic is:

```text
Send patrol goal
→ Nav2 accepts and executes the goal
→ If Person Detected is received
→ Cancel current Nav2 goal
→ Publish zero velocity
→ Wait until status returns to Normal
→ Continue the current or next patrol goal
```

### 6.5 `slam/`

This directory stores SLAM and Nav2-related configuration files.

- `safe_slam_launch.py`: Safe SLAM launch file that only starts slam-toolbox, avoiding duplicated chassis or LiDAR startup.
- `WHEELTEC.yaml`: Map configuration file used by Nav2.
- `param_R550A_PLUS_diff_arm.yaml`: Nav2 parameter file for the R550A_PLUS_diff_arm chassis.

### 6.6 `start/`

This directory stores startup scripts for testing and demonstration.

- `start_base.sh`: Starts the chassis control node.
- `start_lidar.sh`: Starts the YDLIDAR X3 node.
- `start_nav.sh`: Starts Nav2.
- `start_demo_prepare.sh`: Starts the base, LiDAR, Nav2, YOLO, Web, and image conversion modules, but does not start patrol immediately.
- `start_patrol_final.sh`: Starts the final patrol script after the initial pose is set in RViz.
- `stop_robot.sh`: Publishes zero velocity to `/cmd_vel` for emergency stop or safe shutdown.

### 6.7 `web/`

This directory stores Web monitoring programs.

- `compressed_to_raw.py`: Converts image topics for Web display.
- `patrol_web.py`: Flask-based Web service that displays live video, patrol status, alarm information, and basic control buttons.

### 6.8 `yolo/`

This directory stores YOLO patrol status processing code.

- `yolo_patrol_node.py`: Subscribes to `/hobot_dnn_detection`, checks whether a `person` object is detected, and publishes `/patrol/status`.

## 7. Quick Start

> The following steps assume that the project has been deployed to the RDK X5 and all ROS2/TROS/WHEELTEC dependencies are already configured.

### 7.1 Grant Execution Permission

From the repository root directory, run:

```bash
chmod +x start/*.sh
chmod +x nav/*.py
chmod +x web/*.py
chmod +x yolo/*.py
```

### 7.2 Start Demo Preparation

Run:

```bash
bash start/start_demo_prepare.sh
```

This script starts:

```text
Chassis control
YDLIDAR X3 LiDAR
Nav2 navigation
YOLO human detection
Patrol status node
Web monitoring service
Image conversion node
```

At this stage, the robot will not start patrol automatically.

### 7.3 Set Initial Pose

Open RViz, load the map, display `/scan`, and set the robot pose using:

```text
2D Pose Estimate
```

Make sure the `/scan` points overlap with the map walls correctly before starting patrol.

### 7.4 Start Final Patrol

Run:

```bash
bash start/start_patrol_final.sh
```

The robot will start the predefined multi-point patrol task.

## 9. Mapping Workflow

To rebuild the map, follow the workflow below:

```text
1. Start the chassis
2. Start the LiDAR
3. Start safe_slam_launch.py
4. Open RViz and display /map, /scan, and TF
5. Move the robot slowly to build the map
6. Save the map as .pgm and .yaml
7. Copy the new map files to the map/ directory and update the Nav2 map configuration
```

The provided `slam/safe_slam_launch.py` only starts slam-toolbox. It is used to avoid serial conflicts caused by launch files that start the chassis, LiDAR, and SLAM at the same time.

## 11. Key ROS2 Topics

| Topic | Message Type | Description |
|---|---|---|
| `/cmd_vel` | `geometry_msgs/msg/Twist` | Chassis velocity control |
| `/scan` | `sensor_msgs/msg/LaserScan` | LiDAR scan data |
| `/map` | `nav_msgs/msg/OccupancyGrid` | Map used by SLAM and Nav2 |
| `/tf` | `tf2_msgs/msg/TFMessage` | Coordinate transforms |
| `/odom` | `nav_msgs/msg/Odometry` | Odometry feedback |
| `/hobot_dnn_detection` | `ai_msgs/msg/PerceptionTargets` | YOLO detection results |
| `/patrol/status` | `std_msgs/msg/String` | Patrol status, such as Normal or Person Detected |

## 12. Safety Notes

1. Before starting patrol, make sure the map, initial pose, and LiDAR direction are correct.
2. The initial pose must be set in RViz using 2D Pose Estimate before Nav2 patrol.
3. Do not rely on `Ctrl+C` alone to stop robot motion. Run `stop_robot.sh` to actively publish zero velocity.
4. Keep enough free space around the robot during testing.
5. Back up the working version before modifying LiDAR direction, TF, maps, or Nav2 parameters.
6. For competition demonstration, it is recommended to freeze all core parameters once the workflow is stable.
